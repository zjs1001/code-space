    # 一、seo是什么

    SEO（Search Engine Optimization）：汉译为搜索引擎优化。是一种方式：利用搜索引擎 (opens new window)的规则提高网站在有关搜索引擎内的自然排名 (opens new window)。目的是让其在行业内占据领先地位，获得品牌 (opens new window)收益。很大程度上是网站经营者的一种商业行为，将自己或自己公司的排名前移。

    优化策略

    • 主题要明确，内容要丰富
    • 引出链接要人气化
    • 关键词设定要突出
    • 网站架构层次要清晰
    • 内容页面容量合理化
    • 网站导航要清晰化
    • 网站发布要更新

    了解更多内容需要自行百度，本文重点为vue项目中如何进行seo优化

    # 二、普通的前端网站优化方案

    1、合理的title、description、keywords

    <title>博客</title>
<meta name="description" content='一名前端工程师，这是我的个人博客，主要分享前端但不限于前端技术'>
<meta name="keywords" content="个人博客,前端,技术,WEB,blog,BLOG,搭建博客,前端技术">
<meta name="anthor" content="张真人">
<meta name="robots" content="博客,前端,blog,个人博客,web,VUE">
<meta name="viewport" content="width=device-width,initial-scale=1.0">

    title： 就是我们看到的网页标题：只强调重点即可，尽量把重要的关键词放在前面，关键词不要重复出现，尽量做到每个页面的

    标题中不要设置相同的内容<p></p> <p><strong>description：</strong> 为对该网页的简要描述：列举出几个页面的重要关键字即可，切记过分堆砌</p> <p><strong>keywords：</strong> 关键词的作用就是告诉搜索引擎，本网页中主要围绕着哪些方面的关键词展开，需要高度概括网页内容，切记不能太长，过分堆砌关键词，每个页面也要有所不同</p> <p>2、语义化书写html代码，符合W3C标准</p> <p>尽量让代码语义化，在适当的位置使用适当的标签，用正确的标签做正确的事。让阅读源码者和“蜘蛛”都一目了然。比如：h1-h6 是用于标题类的，<code><header></header>,<footer></footer>,<nav></nav>,<aside></aside>,<section></section></code>见名知意。 标签是用来设置页面主导航，列表形式的代码使用ul或ol，重要的文字使用strong等。</p> <p>3、img标签应使用 "alt" 属性加以说明</p> <div <p>8.向各大搜索引擎提交收录自己的站点</p> <p>搜索引擎收录了你的网站后，会很大程度上提升网站的排名</p> <p>9、其他</p> <p>少用iframe：iframe中的内容是不会被抓取到的 提高网站速度：这也是搜索引擎排序的一个重要指标 流量：访问你的网站的人越多，排名也会越靠前</p> <h3 id="三、vue项目中如何进行seo优化"><a href="#三、vue项目中如何进行seo优化" class="header-anchor">#</a> 三、vue项目中如何进行seo优化</h3> <p>在此之前先了解vue开发项目的特点，VUE是一个SPA应用，就是只有一个Web页面的应用，是加载单个HTML页面，并在用户与应用程序交互时动态更新该页面的Web应用程序。</p> <p><strong>为何vue不利于SEO？</strong></p> <p>1、vue用js来渲染数据对seo不友好</p> <p>搜索引擎的基础爬虫的原理就是抓取你的url，然后获取你的html源代码并解析。由于vue通过js的数据绑定机制来展示页面数据，爬虫获取到的html是你的模型页面而不是最终数据的渲染页面，所以说用<strong>js来渲染数据对seo并不友好</strong>。</p> <p>2、vue单页页面对seo不友好</p> <p>seo 本质是一个服务器向另一个服务器发起请求，解析请求内容。但一般来说搜索引擎是不回去执行请求到的js的。也就是说，如果一个单页应用，html在服务器端还没有渲染部分数据数据，在浏览器才渲染出数据，而搜索引擎请求到的html是没有渲染数据的。 这样就很不利于内容被搜索引擎搜索到。 所以<strong>服务端渲染就是尽量在服务器发送到浏览器前页面上就是有数据的</strong>。</p> <p>3、vue数据逻辑操作不合理</p> <p>一般的数据逻辑操作是放在后端的。排序这个如果仅仅是几条数据，前后端排序开起来是一样的，如果是有1000条数据，前端要排序就要都请求过来。这样显然是不合理的。</p> <h4 id="解决方案"><a href="#解决方案" class="header-anchor">#</a> 解决方案</h4> <p>SSR服务器渲染（vue官网提到可深入了解）； 静态化（使用nuxtJS（vue官网提到可深入了解））； 预渲染prerender-spa-plugin； 使用Phantomjs针对爬虫做处理。</p>
